# AI Metrics Calculator

Репозиторий: [github.com/vitalikminaeff/AI_non-LLM_Metrics_Calculator](https://github.com/vitalikminaeff/AI_non-LLM_Metrics_Calculator)

**Клиентский дашборд для оценки качества поиска и классификации** — считает Accuracy, Precision, Recall, F1, Reciprocal Rank (RR) и Mean Reciprocal Rank (MRR) по введённым данным конфьюжн-матрицы и рангу первого релевантного результата. Работает полностью в браузере, без сервера и без LLM.

| | |
|--|--|
| **Файл приложения** | `AI_non LLM_calc-v1.0.0.html` |
| **Стек** | HTML5, CSS, ванильный JavaScript |

## Возможности

- **Ввод данных:** TP, FP, FN, TN, контекст K, ранг первого релевантного ответа (для RR), опционально имя модели и промпта.
- **Дашборд метрик:** значения округляются до двух знаков; для Precision/Recall есть подписи по K.
- **История расчётов:** сохранение последних запусков в **localStorage** (лимит **30** записей), поиск по model / prompt / дате, фильтр по model, удаление записей, раскрываемые матрицы ошибок.
- **Статистика:** средние Precision, Recall, F1 по отфильтрованным run; MRR по всем run и по выборке; лучший/худший run по RR.
- **График MRR** по сохранённым run с легендой по моделям и настройками отображения (сетка, средняя линия, точки, подсказки).
- **Quick MRR Calculator** — быстрый расчёт MRR по списку рангов (через запятую).
- **Тёмная тема** переключается и запоминается в браузере.

## Запуск

1. Клонируйте репозиторий или скачайте файл `AI_non LLM_calc-v1.0.0.html`.
2. Откройте файл двойным щелчком или через меню браузера **«Открыть файл»**.

## Приватность

Все данные остаются **на вашем устройстве**. История хранится только в **localStorage** этого браузера; при очистке данных сайта или смене браузера история пропадает.

## Формулы (кратко)

| Метрика | Формула |
|--------|---------|
| Accuracy | \((TP + TN) / (TP + TN + FP + FN)\) |
| Precision | \(TP / (TP + FP)\) |
| Recall | \(TP / (TP + FN)\) |
| F1 | \(2PR / (P + R)\) |
| RR | \(1/\text{rank}\) при \(\text{rank} > 0\), иначе 0 |
| MRR | среднее RR по сохранённым run (или по введённым рангам в Quick MRR) |

При нулевых знаменателях в коде используется безопасное деление (результат 0 там, где это предусмотрено логикой приложения).
